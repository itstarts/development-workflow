# Fictional direct chapter writing app

This fixture represents a browser-based writing product with a small API and local persistence layer. Existing unit, component, and API tests remain the main regression suite. Browser E2E is intentionally reserved for a minimal set of cross-layer release-critical paths because it is slower and more expensive to maintain.

The target users are long-form fiction writers. A product owner can recruit target writers for moderated acceptance sessions and retain task notes, participant judgments, and screenshots as review evidence.
