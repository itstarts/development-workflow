# 订单导出筛选技术规格

## 目标

为内部 `export_orders` 增加可选审批状态过滤，同时保留无筛选参数时的现有结果。

## 非目标

不改变状态枚举、权限、数据模型、导出格式、公共 API 或外部状态。

## 当前证据

`src/orders.py` 提供内部订单创建和 CSV 导出模型；`tests/test_orders.py` 覆盖现有无筛选导出。

## 行为与边界

过滤值缺失时导出全部输入订单；过滤值存在时只导出 `status` 完全匹配的订单。输入列表和订单对象保持不变。

## 组件与控制流

只修改内部 `export_orders` 过滤逻辑并补定向单元测试，不引入新组件或依赖。

## API 与技术接口

内部函数增加可选 `status=None` 参数；无公开接口变化。

## 命令结果与失败矩阵

该只读内部函数不改变持久化状态或触发异步工作，因此不需要状态变更结果矩阵。

## 数据模型与实体关系

不变。

## 数据库事务与锁语义

不适用；该 fixture 不使用数据库。

## 状态转换、迁移边界与一致性

不改变订单状态，不涉及迁移、事务或共享状态。

## 错误与不确定性

按精确字符串比较现有状态；不新增校验或降级行为。

## 保证与测试追踪

- `G-01`：无筛选保持全部结果；`tests/test_orders.py::OrderTests::test_export_preserves_existing_orders`。
- `G-02`：有筛选只保留匹配状态；新增 `tests/test_orders.py::OrderTests::test_export_filters_by_status`。

## 验收类型与证据

定向单元测试足以覆盖内部纯函数行为；不需要关键 E2E 或主观人工验收。

## 测试与文档

运行 `python3 -m unittest tests.test_orders`；无需产品文档变化。

## 验收标准

- 无筛选结果与当前行为一致。
- 有筛选时只返回匹配订单。
- 输入对象不被修改。
