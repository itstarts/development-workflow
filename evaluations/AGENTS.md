# AGENTS.md — evaluations

## 适用范围

适用于版本化的 skill baseline、GREEN 前向测试和集成评估证据。

## 版本化证据

- 脱敏后的固定场景、判据和选中证据必须版本化，确保 GitHub review 可以重放结论。
- 每个 skill 使用独立目录，至少区分 `cases/`、`baseline/`、`green/` 和结构化结果。
- baseline 使用无目标 skill 的全新代理；GREEN 使用不同的全新代理和完全相同的场景。
- 保存原始可审查输出时，不向测试代理暴露 expected、失败分析或实现结论。
- 修复后只保留最终选中输出；被替换的原始材料留在被忽略的 `work/`，无需建立逐次 attempt 审计链。

## 隐私与本地证据

- 真实 task/thread 标识符、标题、预览、凭证、生产数据和未经脱敏的用户内容不得进入版本库。
- 只能保存在被忽略的 `work/` 中的证据，在版本化结果中记录必要的类型、结论和限制即可，不要求文件哈希。
- 不得用静态合同或回复文本冒充工具 trace；替代验收必须记录用户授权和保证差异。

## 完成门

- 判据在 GREEN 前固定；需求变化时直接更新判据并重新运行受影响场景，不维护哈希历史。
- 失败场景先形成 RED，再做最小修复。
- 自动化结果、前向结果、评审结论和残留风险互相一致。
- evidence 文件不得包含本机绝对路径；仓库相对路径用于指向版本化材料。
